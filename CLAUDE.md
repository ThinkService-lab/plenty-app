# CLAUDE.md — Plenty App

AI assistant context for this codebase. Read this before making any changes.

---

## What this app is

Plenty is a single-page AI meal suggestion app. Users enter ingredients they have at home and receive 3 personalised meal suggestions generated by Claude Haiku. The tone is empowering and abundant — the word "only" is banned from the AI prompt.

---

## Architecture — one file rule

**The entire frontend lives in `index.html`.** There is no build step, no framework, no bundler. Every CSS rule, HTML element, and JS function is in this one file. Do not split it.

The backend is three Vercel serverless functions in `/api/`:
- `meals.js` — proxies the Anthropic API call
- `subscribe.js` — saves emails to Google Sheets
- `photo.js` — unused, leave it

---

## Critical JS rules

**All onclick functions must be on `window.*`** — functions are defined inside a `<script>` block, not globally. Any new function called via `onclick="..."` in HTML must be assigned as `window.functionName = function() {}` or it will throw `undefined is not a function`.

**Do not use `innerHTML =` on user-controlled input** — the security hook will block your edit. Use the template literal pattern already established for `card` markup (set via `.innerHTML` on createElement'd divs with server-returned data, which is acceptable here).

**JS template string edits** — the meal card scaffold is built inside `renderResults()` as template literals. Only touch the HTML scaffold strings (what elements exist), never the data logic, API calls, event handlers, or function signatures.

## CSS design system

Token names (defined in `:root`):
```
--cream, --warm-white, --earth, --terracotta, --sage, --gold, --light-sage, --muted
--shadow-float, --shadow-hover, --shadow-card
--hero-gradient
--glass-bg, --glass-border
--spring, --ease-out-expo
```

Font stack: `Playfair Display` (serif, headings, weights 400/700/800) + `DM Sans` (sans-serif, body, weights 300/400/500).

**Do not add new fonts.** Both are already loaded from Google Fonts.

## Meal card colour system

Three cards use **triadic colour harmony** (~120° apart on the colour wheel), applied via `styleIndex = mealIndex % 3`:
- Index 0: `linear-gradient(135deg, #C4622D 0%, #D4A847 100%)` — Terracotta → Gold
- Index 1: `linear-gradient(135deg, #2D7A5A 0%, #5AAF85 100%)` — Forest → Sage
- Index 2: `linear-gradient(135deg, #6B3A8F 0%, #C4622D 100%)` — Plum → Terracotta

Meal card header text is always white (`#fff`) — do not change this.

## Key behaviours

- **Form hides on results** — `#formSection` is hidden when `resultsSection` becomes active. `resetApp()` restores it.
- **Loading state** — aurora blobs + dual rings + shimmer skeleton cards. Class `.loading-state.active` shows it.
- **Slow load fallback** — after 15s, `.loading-text` text is replaced with a reassurance message.
- **Rate meals influence next generation** — liked/disliked meals are tracked in `ratingContext` and injected into the next prompt.
- **Health conditions** — stored in `selectedConditions` (Set). Rules are injected into the AI prompt via `buildHealthRules()`.

---

## What NOT to change without care

- `api/meals.js` — the `charCodeAt()` control character cleaning is intentional; regex with literal newlines broke Vercel compilation silently
- `max_tokens: 1800` — do not reduce; health condition prompts need the headroom
- `claude-haiku-4-5-20251001` — intentional; sonnet times out on Vercel free tier
- Feedback POST goes directly to Google Apps Script URL from the frontend — there is no Vercel function for it

## Local dev

```bash
vercel dev          # starts at http://localhost:3000
```

Requires `.env.local` with `ANTHROPIC_API_KEY=...` in the project root.

---

## Deployment

Push to `main` → Vercel auto-deploys. No build step needed.

**Always work on a branch first.** The app is live on Vercel from `main`.

```bash
git checkout -b my-feature
# make changes, test with vercel dev
git push origin my-feature
# when ready:
git checkout main && git merge my-feature && git push origin main
```
